The short version: your typing stays on your device, and nothing is ever sent anywhere except the single sentence you explicitly ask to have corrected — sentence text is never stored or logged by us (the server keeps only the anonymous request metadata described below).

Монгол Keyboard is an iOS keyboard that converts Mongolian Latin transliteration to Cyrillic on your device, with an optional AI grammar-correction feature. This policy describes exactly what the app does with your data.

What the keyboard does with your typing

  • All transliteration (Latin → Cyrillic) happens entirely on your device. Keystrokes are never logged or transmitted.
  • When you pick a suggestion from the candidate bar, that single word pair (what you typed → what you chose) is saved to a small dictionary in the app's private container on your device, so it can be offered first next time. It never leaves the device, is capped in size, and can be cleared any time in the app. Saving these picks requires the Allow Full Access permission (iOS only allows a keyboard to write to the app's shared container with it); without the permission the keyboard works identically but doesn't remember your picks.
  • The keyboard contains no analytics, no advertising SDKs, and no trackers of any kind.

The ✨Засах (grammar fix) feature

  • Only when you explicitly tap the Засах key, the current sentence (at most 500 characters) is sent over an encrypted HTTPS connection for grammar correction, and the corrected sentence is returned.
  • Depending on how the app is set up, that request goes to one of three places:
    • Built-in server (on releases that include it) — the app's own correction server (keyboard-api.zon.mn), which forwards the sentence to the Anthropic API. The server holds the API key, enforces rate limits, and never stores or logs sentence text. The request also carries a random device identifier (a locally generated UUID) used only for rate limiting — it is not linked to your name, account, or anything else about you. There are no user accounts. You can turn the built-in server off in the app.
    • Custom server mode — the same request format, but to a server whose URL you configured yourself. What that server does with the request is up to its operator — running your own copy of the app's open-source proxy gives you the same behavior as the built-in server; the guarantees in this policy do not extend to arbitrary third-party servers you point the app at.
    • Direct mode — with no server configured, straight to the Anthropic API (api.anthropic.com), authenticated with your own API key.
  • Nothing is sent while you type. The request contains only that one sentence — no conversation history, no location, and no identifiers beyond the anonymous UUID described above.
  • The app does not store the sentences it corrects. Processing by Anthropic is governed by Anthropic's privacy policy; with standard API settings Anthropic does not use API inputs to train its models. Anthropic ordinarily retains API inputs and outputs for up to 30 days; content flagged by its safety systems can be retained longer (up to two years), and retention may be extended where the law requires — see Anthropic's data-retention policy for the authoritative terms.
  • So that corrections respond quickly, when the keyboard opens with the AI feature enabled, Full Access granted, and a correction server configured (the built-in or a custom one), it may make one empty warm-up connection to that server — a request to its public status route that carries no text, no typing, no API key, and no identifiers. Its only purpose is having the encrypted connection ready before you tap Засах. Turning ✨ AI засвар off in the app disables these connections together with the feature. In direct mode (your own API key, no server) there is no warm-up: the keyboard contacts the Anthropic API only for a correction you asked for.
  • If the AI feature is turned off — or Full Access is not granted — the keyboard makes no network connections at all.

Your API key

  • In the built-in-server setup no API key exists on your device at all — most people never need this section.
  • An API key you optionally enter for direct mode is stored only on your device, in the app's private App Group container, so the keyboard extension can read it.
  • It is never transmitted anywhere except directly to api.anthropic.com to authenticate your own requests, and it never appears in logs.

iOS "Allow Full Access"

iOS requires the Allow Full Access permission for any keyboard that makes network requests. Монгол Keyboard uses this permission for exactly two things: the Засах feature (the correction call and its empty warm-up connection, both described above), and saving your candidate-bar picks to the on-device dictionary described above (never transmitted). Typing and transliteration work fully without it — the keyboard just won't remember your picks, and Засах is unavailable.

Data we collect

The app has no accounts, and we never receive your name, email, contacts, location, or anything else about you. When you tap Засах with the built-in server active, the application payload contains exactly two things: the sentence you asked to correct, and the app's random device UUID. Like any internet request, it also carries network-level metadata (notably your IP address and connection details): the built-in server sits behind the Cloudflare network, which processes that metadata transiently to route and protect traffic under Cloudflare's privacy policy; our server itself does not use or store IP addresses. The server processes the request in memory — the sentence is forwarded for correction and then discarded. What it keeps: anonymous request counts per device UUID for the current day (rate limiting), and standard operational logs containing request metadata only — a timestamp, a truncated prefix of the device UUID, the input length, the response status, and the processing time. Sentence text is never written to logs or storage. In Apple's App Privacy terms, the sentence and the device UUID count as collected data "linked" to a device identifier — which is why the App Store label lists them — but they map to no account and no personal identity. In direct mode we operate nothing at all — requests go from your device to Anthropic and we have no way to see them. (A custom server's logging is under its operator's control — see above.)

Children's privacy

The app has no accounts, no ads, no tracking, and is not directed at children. The only data processed is what the Засах feature sends, exactly as described above — the same for every user, including children.
